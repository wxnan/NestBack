## WebDAV备份配置

### 什么是WebDAV？

WebDAV（Web-based Distributed Authoring and Versioning）是一种基于HTTP协议的网络文件访问协议，支持用户通过网络访问和管理远程服务器上的文件。大多数主流云盘服务（如坚果云、阿里云盘等）都支持WebDAV协议。

### 为什么需要WebDAV备份？

- **数据安全**：防止本地数据丢失
- **跨设备访问**：通过云盘访问备份数据
- **加密传输**：支持AES加密，保护隐私

### 配置步骤

#### 1. 选择WebDAV服务提供商

推荐使用以下支持WebDAV的服务：

| 服务商 | 特点 | 免费额度 |
|--------|------|----------|
| 坚果云 | 国内访问稳定，支持WebDAV | 每月1G上传流量 |
| 阿里云盘 | 容量大，需第三方工具 | 需配合WebDAV插件 |
| Seafile | 开源自建，适合极客 | 完全自控 |

#### 2. 获取WebDAV信息

**坚果云示例**：
1. 登录坚果云网页版
2. 点击右上角用户名 > 账户信息
3. 点击"安全选项"
4. 在"第三方应用管理"中创建应用
5. 获取应用密码

#### 3. 在应用中配置

1. 打开归巢应用
2. 进入"我的" > "数据管理" > "备份设置"
3. 填写配置信息：

```
服务器地址：https://dav.jianguoyun.com/dav/
用户名：your_email@example.com
密码：your_app_password（应用密码，不是登录密码）
备份路径：/nestback_backup/
加密密钥：（建议填写，用于AES加密备份数据）
```

4. 点击"测试连通性"
5. 测试成功后，点击"立即备份"

#### 4. 配置加密（推荐）

开启加密功能后，备份数据会以AES加密格式存储，即使云盘被他人访问也无法查看内容。

```
加密密钥设置建议：
- 长度至少12位
- 包含大小写字母、数字、特殊字符
- 不要使用与其他网站相同的密码
- 建议使用密码管理器保存
- ⚠️ 请务必记住加密密钥，丢失后无法恢复数据
```

#### 5. 高级备份功能

- **按家庭备份**：备份文件按家庭名称分类存储，便于管理多个家庭的数据
- **多家庭恢复**：可从备份中选择不同家庭的数据进行恢复，支持跨家庭恢复
- **设置备份**：支持备份AI设置、扫码设置等配置数据，方便在新设备上快速恢复配置
- **手动备份**：需要时手动点击"立即备份"
- **自动备份**：可设置每日自动备份（开发中）

### 常见问题

**Q: 测试连通性失败？**
A: 检查以下几点：
- 服务器地址是否正确（包括尾部斜杠）
- 用户名密码是否正确（坚果云需使用应用密码）
- 网络连接是否正常
- 云盘服务是否正常

**Q: 备份文件在哪里？**
A: 备份文件会保存在你设置的"备份路径"下，按家庭名称分目录存储，文件名为`nestback_backup_YYYYMMDD.zip`。

**Q: 如何恢复备份？**
A: 在备份设置页面，点击"恢复备份"，选择要恢复的备份文件即可。支持恢复指定家庭的数据，也可跨家庭恢复。

**Q: 忘记加密密钥怎么办？**
A: 加密密钥无法找回，请务必妥善保存。建议使用密码管理器保存密钥。

## Supabase云同步配置

> ⚠️ **注意**：Supabase云同步功能目前仍在开发中，以下配置说明仅供参考，实际功能可能尚未完全可用。

### 什么是Supabase？

Supabase是一个开源的Firebase替代品，提供实时数据库、身份认证、文件存储等功能。在归巢应用中，我们使用Supabase实现多设备数据同步和多成员协同。

### 配置步骤

#### 1. 创建Supabase项目

1. 访问 [supabase.com](https://supabase.com)
2. 注册并登录账号
3. 点击"New Project"创建新项目
4. 设置项目名称、数据库密码等
5. 等待项目创建完成（通常需要几分钟）

#### 2. 获取项目配置信息

1. 进入项目控制台
2. 点击左侧"Settings" > "API"
3. 找到并复制以下信息：
   - **Project URL**：如 `https://xxxxx.supabase.co`
   - **anon/public Key**：如 `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`

#### 3. 在应用中配置

1. 打开归巢应用
2. 进入"我的" > "数据管理" > "同步设置"
3. 填写配置信息：

```
URL：https://xxxxx.supabase.co
API Key：eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

4. 点击"测试连通性"

#### 4. 创建数据库表

1. 在Supabase控制台，点击左侧"SQL Editor"
2. 复制以下SQL脚本并执行：

```sql
-- 创建房屋表
CREATE TABLE IF NOT EXISTS houses (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  is_default BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建空间表
CREATE TABLE IF NOT EXISTS spaces (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  icon TEXT,
  image_path TEXT,
  parent_id TEXT REFERENCES spaces(id) ON DELETE CASCADE,
  type TEXT NOT NULL DEFAULT 'container',
  position TEXT,
  default_category_id TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建分类表（全局共享，无house_id外键）
CREATE TABLE IF NOT EXISTS categories (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  icon TEXT,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建二级分类表
CREATE TABLE IF NOT EXISTS subcategories (
  id TEXT PRIMARY KEY,
  category_id TEXT NOT NULL REFERENCES categories(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建标签表（全局共享）
CREATE TABLE IF NOT EXISTS tags (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建属性表
CREATE TABLE IF NOT EXISTS attributes (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  type TEXT NOT NULL,
  hint TEXT,
  options TEXT,
  required BOOLEAN DEFAULT false,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建分类属性关联表
CREATE TABLE IF NOT EXISTS category_attributes (
  category_id TEXT NOT NULL REFERENCES categories(id) ON DELETE CASCADE,
  attribute_id TEXT NOT NULL REFERENCES attributes(id) ON DELETE CASCADE,
  sort_order INTEGER DEFAULT 0,
  PRIMARY KEY (category_id, attribute_id)
);

-- 创建物品表
CREATE TABLE IF NOT EXISTS items (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  space_id TEXT NOT NULL REFERENCES spaces(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  quantity INTEGER DEFAULT 1,
  unit TEXT DEFAULT '个',
  price REAL,
  production_date TIMESTAMPTZ,
  shelf_life INTEGER,
  expire_date TIMESTAMPTZ,
  category TEXT,
  category_id TEXT,
  subcategory_id TEXT,
  tags TEXT,
  image_path TEXT,
  note TEXT,
  custom_attributes TEXT,
  creator_id TEXT NOT NULL,
  modifier_id TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建物品属性关联表
CREATE TABLE IF NOT EXISTS item_attributes (
  item_id TEXT NOT NULL REFERENCES items(id) ON DELETE CASCADE,
  attribute_id TEXT NOT NULL REFERENCES attributes(id) ON DELETE CASCADE,
  value TEXT,
  PRIMARY KEY (item_id, attribute_id)
);

-- 启用RLS（行级安全策略）
ALTER TABLE houses ENABLE ROW LEVEL SECURITY;
ALTER TABLE spaces ENABLE ROW LEVEL SECURITY;
ALTER TABLE items ENABLE ROW LEVEL SECURITY;
ALTER TABLE categories ENABLE ROW LEVEL SECURITY;
ALTER TABLE tags ENABLE ROW LEVEL SECURITY;

-- 创建RLS策略
CREATE POLICY "Users can only access their own houses" ON houses
  FOR ALL USING (true);

CREATE POLICY "Users can only access spaces in their houses" ON spaces
  FOR ALL USING (true);

CREATE POLICY "Users can only access items in their houses" ON items
  FOR ALL USING (true);

CREATE POLICY "Users can only access categories in their houses" ON categories
  FOR ALL USING (true);

CREATE POLICY "Users can only access tags in their houses" ON tags
  FOR ALL USING (true);
```

3. 点击"Run"执行SQL

#### 5. 测试RLS

执行以下SQL验证RLS是否生效：

```sql
-- 应该返回0行（因为还没有数据）
SELECT * FROM houses LIMIT 10;
```

## AI功能配置

### AI功能简介

归巢应用支持两种AI功能：
1. **AI识图**：拍照自动识别物品信息（名称、分类、描述等）
2. **AI聊天**：通过语音或文字描述添加物品

应用采用**提供商 + 模型**管理系统，支持灵活配置多个AI服务商和模型，并内置免费试用Key。

### 支持的AI服务

| 服务商 | 特点 | 价格 | 推荐指数 |
|--------|------|------|----------|
| 智谱AI | 部分flash模型永久免费，仅限制并发数，支持AI识图 | 免费/按量 | ⭐⭐⭐ |
| ModelScope | 免费额度较多，2000次/天，支持AI识图 | 免费/按量 | ⭐⭐⭐ |
| 硅基流动 | 部分小模型永久免费，1000 RPM | 免费/按量 | ⭐⭐ |
| OpenAI | GPT-4o视觉能力强 | 按量计费 | ⭐⭐ |
| 本地Ollama | 完全免费，隐私保护 | 免费 | ⭐ |
| 其他OpenAI兼容API | 灵活适配 | 取决于服务商 | ⭐⭐ |

### 配置步骤

#### 1. 快速开始（使用内置免费Key）

应用内置免费试用AI Key，无需任何配置即可使用AI功能：

1. 进入"我的" > "数据管理" > "AI设置"
2. 在"默认模型"中分别选择默认聊天模型和默认识图模型
3. 返回录入页面即可使用AI识图和AI聊天

> 💡 **提示**：内置免费Key可能随时失效，建议自行配置AI服务以获得稳定体验。

#### 2. 添加AI提供商

1. 进入"我的" > "数据管理" > "AI设置"
2. 在"AI提供商"区域，点击"添加提供商"
3. 填写提供商信息：
   - **名称**：自定义名称（如"智谱AI"、"我的Ollama"等）
   - **Base URL**：AI服务地址
   - **API Key**：访问密钥
   - **API路径**：（可选）默认为 `/chat/completions`
   - **自定义请求头**：（可选）JSON格式的额外请求头
4. 点击保存

#### 3. 添加AI模型

1. 在AI设置页面，点击已创建的提供商
2. 点击"添加模型"
3. 填写模型信息：
   - **模型ID**：服务商提供的模型标识（如 `gpt-4o`、`glm-4v-flash` 等）
   - **名称**：自定义显示名称
   - **类型**：聊天模型 或 识图模型
4. 启用模型
5. 在"默认模型"区域选择该模型作为默认

#### 4. 设置默认模型

在使用AI识图和AI聊天前，必须设置默认模型：

1. 进入AI设置页面
2. 在"默认模型"区域：
   - **默认聊天模型**：选择用于AI聊天录入的模型
   - **默认识图模型**：选择用于AI识图录入的模型
3. 未设置默认模型时，对应功能会提示前往设置

## 扫码API配置

### 扫码功能说明

扫码API用于根据商品条形码查询商品信息（如名称、品牌、图片等），实现快速录入。应用使用ML Kit进行本地条码识别，再调用商品API查询商品详情。

### 支持的扫码API服务

| 服务商 | 特点 | 免费额度 | 推荐指数 |
|--------|------|----------|----------|
| apizero | 无Key也可使用，小部分有图 | 200次/天 | ⭐⭐⭐ |
| apizero-pro | 无Key也可使用，大部分有图，官方数据库 | 20次/天 | ⭐⭐⭐ |
| apibyte | 需要API Key，商品信息较少 | 100次/天 | ⭐⭐ |
| rollapi | 免费额度多，无图 | 1000次/天 | ⭐⭐⭐ |

### 配置步骤

#### 1. 选择扫码服务商

1. 进入"我的" > "数据管理" > "扫码设置"
2. 在服务商列表中选择一个扫码API提供商

#### 2. apizero / apizero-pro 配置

无需API Key即可使用：

1. 选择"apizero"或"apizero-pro"
2. 无需填写额外配置
3. 点击"测试条码查询"验证

#### 3. apibyte 配置

1. 注册apibyte账号并获取API Key
2. 选择"apibyte"
3. 填写API Key
4. 点击"测试条码查询"验证

#### 4. rollapi 配置

1. 注册rollapi账号并获取App ID和App Secret
2. 选择"rollapi"
3. 填写App ID和App Secret
4. 点击"测试条码查询"验证

### 测试与使用

1. 配置完成后，点击"测试条码查询"
2. 输入一个已知的条形码进行测试（如 `6921168509256`）
3. 如果返回正确的商品信息，说明配置成功
4. 在"录入"页面点击"扫码录入"使用

### 常见问题

**Q: 扫码查询失败？**
A: 检查以下几点：
- API服务商是否正确选择
- API Key是否正确（apibyte/rollapi）
- 网络连接是否正常
- 条形码是否清晰可读

**Q: 条码查询未找到商品？**
A: 说明该条码商品信息未在数据库中。建议更换API服务商尝试，或使用手动录入。

**Q: 只填充了部分商品信息？**
A: 可能是数据库中缺少该条码的详细商品信息。建议更换API服务商尝试。
