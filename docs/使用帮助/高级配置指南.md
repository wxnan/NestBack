## WebDAV备份配置

### 什么是WebDAV？

WebDAV（Web-based Distributed Authoring and Versioning）是一种基于HTTP协议的网络文件访问协议，支持用户通过网络访问和管理远程服务器上的文件。大多数主流云盘服务（如坚果云、阿里云盘等）都支持WebDAV协议。

### 为什么需要WebDAV备份？

- **数据安全**：防止本地数据丢失
- **跨设备访问**：通过云盘访问备份数据
- **加密传输**：支持AES加密，保护隐私

### 配置步骤

#### 1. 选择WebDAV服务提供商

推荐使用以下支持WebDAV的服务：

| 服务商 | 特点 | 免费额度 |
|--------|------|----------|
| 坚果云 | 国内访问稳定，支持WebDAV | 每月1G上传流量 |
| 阿里云盘 | 容量大，需第三方工具 | 需配合WebDAV插件 |
| Seafile | 开源自建，适合极客 | 完全自控 |

#### 2. 获取WebDAV信息

**坚果云示例**：
1. 登录坚果云网页版
2. 点击右上角用户名 > 账户信息
3. 点击"安全选项"
4. 在"第三方应用管理"中创建应用
5. 获取应用密码

#### 3. 在应用中配置

1. 打开归巢应用
2. 进入"我的" > "数据管理" > "WebDAV设置"
3. 填写配置信息：

```
服务器地址：https://dav.jianguoyun.com/dav/
用户名：your_email@example.com
密码：your_app_password（应用密码，不是登录密码）
备份路径：/NestBack/
加密密钥：（可选，建议填写）
```

4. 点击"测试连通性"
5. 测试成功后，点击"立即备份"

#### 4. 配置加密（可选）

开启加密功能后，备份数据会以AES-256加密格式存储，即使云盘被他人访问也无法查看内容。

```
加密密钥设置建议：
- 长度至少12位
- 包含大小写字母、数字、特殊字符
- 不要使用与其他网站相同的密码
- 建议使用密码管理器保存
```

#### 5. 备份频率设置

- **手动备份**：需要时手动点击"立即备份"
- **自动备份**：可设置每日自动备份（开发中）

### 常见问题

**Q: 测试连通性失败？**
A: 检查以下几点：
- 服务器地址是否正确（包括尾部斜杠）
- 用户名密码是否正确
- 网络连接是否正常
- 云盘服务是否正常

**Q: 备份文件在哪里？**
A: 备份文件会保存在你设置的"备份路径"下，文件名为`nestback_backup_YYYYMMDD.zip`。

**Q: 如何恢复备份？**
A: 在WebDAV设置页面，点击"恢复备份"，选择要恢复的备份文件即可。

## Supabase云同步配置

### 什么是Supabase？

Supabase是一个开源的Firebase替代品，提供实时数据库、身份认证、文件存储等功能。在归巢应用中，我们使用Supabase实现多设备数据同步和多成员协同。

### 配置步骤

#### 1. 创建Supabase项目

1. 访问 [supabase.com](https://supabase.com)
2. 注册并登录账号
3. 点击"New Project"创建新项目
4. 设置项目名称、数据库密码等
5. 等待项目创建完成（通常需要几分钟）

#### 2. 获取项目配置信息

1. 进入项目控制台
2. 点击左侧"Settings" > "API"
3. 找到并复制以下信息：
   - **Project URL**：如 `https://xxxxx.supabase.co`
   - **anon/public Key**：如 `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`

#### 3. 在应用中配置

1. 打开归巢应用
2. 进入"我的" > "数据管理" > "Supabase设置"
3. 填写配置信息：

```
URL：https://xxxxx.supabase.co
API Key：eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

4. 点击"测试连通性"

#### 4. 创建数据库表

1. 在Supabase控制台，点击左侧"SQL Editor"
2. 复制以下SQL脚本并执行：

```sql
-- 创建房屋表
CREATE TABLE IF NOT EXISTS houses (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  is_default BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建空间表
CREATE TABLE IF NOT EXISTS spaces (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  parent_id TEXT REFERENCES spaces(id) ON DELETE CASCADE,
  type TEXT NOT NULL DEFAULT 'container',
  icon TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建物品表
CREATE TABLE IF NOT EXISTS items (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  space_id TEXT NOT NULL REFERENCES spaces(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  quantity INTEGER DEFAULT 1,
  price REAL,
  category TEXT,
  tags TEXT,
  image TEXT,
  barcode TEXT,
  brand TEXT,
  color TEXT,
  min_stock INTEGER DEFAULT 1,
  storage_method TEXT,
  production_date TIMESTAMPTZ,
  expiry_date TIMESTAMPTZ,
  reminder_days INTEGER DEFAULT 7,
  opened_date TIMESTAMPTZ,
  purchase_date TIMESTAMPTZ,
  warranty_period INTEGER,
  warranty_expire TIMESTAMPTZ,
  notes TEXT,
  creator_id TEXT,
  modifier_id TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建分类表
CREATE TABLE IF NOT EXISTS categories (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  icon TEXT,
  attributes TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 创建标签表
CREATE TABLE IF NOT EXISTS tags (
  id TEXT PRIMARY KEY,
  house_id TEXT NOT NULL REFERENCES houses(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  color TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 启用RLS（行级安全策略）
ALTER TABLE houses ENABLE ROW LEVEL SECURITY;
ALTER TABLE spaces ENABLE ROW LEVEL SECURITY;
ALTER TABLE items ENABLE ROW LEVEL SECURITY;
ALTER TABLE categories ENABLE ROW LEVEL SECURITY;
ALTER TABLE tags ENABLE ROW LEVEL SECURITY;

-- 创建RLS策略
CREATE POLICY "Users can only access their own houses" ON houses
  FOR ALL USING (true);

CREATE POLICY "Users can only access spaces in their houses" ON spaces
  FOR ALL USING (true);

CREATE POLICY "Users can only access items in their houses" ON items
  FOR ALL USING (true);

CREATE POLICY "Users can only access categories in their houses" ON categories
  FOR ALL USING (true);

CREATE POLICY "Users can only access tags in their houses" ON tags
  FOR ALL USING (true);
```

3. 点击"Run"执行SQL

#### 5. 测试RLS

执行以下SQL验证RLS是否生效：

```sql
-- 应该返回0行（因为还没有数据）
SELECT * FROM houses LIMIT 10;
```

## AI功能配置

### AI功能简介

归巢应用支持两种AI功能：
1. **AI识图**：拍照自动识别物品信息
2. **AI聊天**：通过语音或文字描述添加物品

### 支持的AI服务

| 服务商 | 特点 | 价格 |
|--------|------|------|
| OpenAI | GPT-4o视觉能力强 | 按量计费 |
| Anthropic Claude | Claude 3.5 Sonnet | 按量计费 |
| 本地Ollama | 完全免费，隐私保护 | 免费 |
| 硅基流动 | 国内可用，价格实惠 | 按量计费 |
| 其他OpenAI兼容API | 灵活适配 | 取决于服务商 |

### 配置步骤

#### 1. 选择AI服务商

**推荐方案对比**：

| 方案 | 适用场景 | 配置难度 | 费用 |
|------|----------|----------|------|
| OpenAI官方API | 有海外网络条件 | ⭐⭐ | 较高 |
| 硅基流动 | 国内用户首选 | ⭐⭐ | 中等 |
| 本地Ollama | 注重隐私/极客 | ⭐⭐⭐ | 免费 |
| 自建API代理 | 企业用户 | ⭐⭐⭐⭐ | 可控 |

#### 2. OpenAI配置示例

1. 访问 [platform.openai.com](https://platform.openai.com)
2. 注册账号并充值
3. 进入"API Keys"页面
4. 创建新的API Key
5. 在应用中配置：

```
Base URL：https://api.openai.com/v1
API Key：sk-...
Vision模型：gpt-4o
Chat模型：gpt-4o
```

#### 3. 硅基流动配置示例

1. 访问 [siliconflow.cn](https://siliconflow.cn)
2. 注册并登录
3. 进入个人中心 > API密钥
4. 创建新的API Key
5. 在应用中配置：

```
Base URL：https://api.siliconflow.cn/v1
API Key：sk-...
Vision模型：Qwen/Qwen2-VL-72B-Instruct
Chat模型：Qwen/Qwen2.5-7B-Instruct
```

#### 4. 本地Ollama配置示例

1. 安装Ollama（[官网](https://ollama.com)）
2. 下载模型：
```bash
# 下载视觉模型
ollama pull llava

# 下载聊天模型
ollama pull qwen2.5:7b
```
3. 启动Ollama服务（默认端口11434）
4. 在应用中配置：

```
Base URL：http://localhost:11434/v1
API Key：ollama（任意值，Ollama不需要Key）
Vision模型：llava
Chat模型：qwen2.5:7b
```

### AI功能使用

#### AI识图

1. 进入"录入"页面
2. 点击"AI识图"
3. 拍摄物品照片
4. 系统自动识别并填充物品信息
5. 确认并保存

#### AI聊天

1. 进入"录入"页面
2. 点击"AI聊天"
3. 输入文字描述或按住麦克风语音输入
4. 系统解析描述并填充物品信息
5. 确认并保存

### 费用说明

- AI功能按调用次数和Token数量收费
- 视觉识别每次调用约消耗100k-500k Token
- 聊天功能根据描述长度收费
- 建议设置预算提醒，避免超支

### 隐私说明

- 发送到AI服务的图片会经过压缩处理
- 包含物品信息的文字描述会发送到AI服务
- 建议使用本地Ollama以获得最佳隐私保护

## 扫码API配置

### 扫码功能说明

扫码API用于根据商品条形码查询商品信息（如名称、品牌、图片等），实现快速录入。

### 支持的扫码API服务

| 服务商 | 特点 | 免费额度 |
|--------|------|----------|
| 聚合数据 | 商品数据全面 | 有免费额度 |
| 百度AI | 识别速度快 | 按量计费 |
| 淘宝开放平台 | 商品信息准确 | 需要AppKey |
| 自建服务 | 完全可控 | 免费 |

### 配置步骤

#### 1. 聚合数据配置示例

1. 访问 [dataju.cn](https://www.dataju.cn) 或 [juhe.cn](https://www.juhe.cn)
2. 注册并登录
3. 申请"商品条码查询"API
4. 获取API Key
5. 在应用中配置：

```
API地址：https://api.dataju.cn/api/barcode/query
API Key：your_api_key
```

#### 2. 通用API配置

如果使用其他扫码API服务，配置格式如下：

```
API地址：https://your-api.com/barcode
API Key：your_api_key
请求参数：barcode（条形码自动填充）
响应格式：JSON
```

#### 3. 响应字段映射

系统期望API返回以下字段：

```json
{
  "name": "商品名称",
  "brand": "品牌",
  "image": "商品图片URL",
  "category": "分类",
  "price": "参考价格"
}
```

如果API返回格式不同，可在配置中映射字段。

### 测试与使用

1. 配置完成后，点击"测试条码查询"
2. 输入一个已知的条形码进行测试
3. 如果返回正确的商品信息，说明配置成功
4. 在"录入"页面点击"扫码录入"使用
